## 步骤 2：解决版本兼容问题

### 2.1 Django 版本降级

pip 默认安装了 Django 5.2.14，但该项目基于 Django 3.2 开发，且 **Django 5.x 要求 MySQL 8.0.11+**，当前环境是 MySQL 5.7.40，启动时报错：

```
django.db.utils.NotSupportedError: MySQL 8.0.11 or later is required (found 5.7.40).
```

**解决**：降级到 Django 3.2

```bash
pip install "django>=3.2,<4.0"
```

降级后版本：**Django 3.2.25**，兼容 MySQL 5.7。

### 2.2 PyTorch 模型加载兼容

项目使用了 NCF 推荐模型文件 `utils/recommender/ncf_production.pth`，该文件由旧版 PyTorch（pickle 格式）保存。当前安装的 PyTorch 2.11 默认 `torch.load(weights_only=True)`（安全模式），拒绝加载 pickle 文件，首页报错：

```
UnpicklingError: Weights only load failed.
Unsupported global: GLOBAL numpy.core.multiarray._reconstruct was not an allowed global...
```

**解决**：修改 `e:\MoviesRec\utils\recommender\recommend.py` 第 38 行，添加 `weights_only=False`：

```python
# 修改前
checkpoint = torch.load(MODEL_PATH, map_location="cpu")
# 修改后
checkpoint = torch.load(MODEL_PATH, map_location="cpu", weights_only=False)
```

> 这在开发环境是安全的（模型文件来自项目自身），实际部署时建议用新版 PyTorch 重新训练并导出模型。

# MovieRec 项目功能总结

## 推荐引擎（核心）

系统采用**三阶段推荐策略**，根据用户活跃度自动切换：

| 阶段 | 条件 | 策略 |
|------|------|------|
| 冷启动 | 未登录 | 热门电影排行（按评分人数 + 均分） |
| 过渡期 | 已登录 + 操作 < 30 次 | 基于偏好类型的混合推荐（70% 匹配 + 30% 探索） |
| 个性化 | 已登录 + 操作 >= 30 次 | NCF 神经网络协同过滤（加权用户向量 + 余弦相似度） |

### 推荐策略详解

**策略一：热门推荐 `get_hot_movies()`**

- 统计 Rating 表中的评分数量和平均分
- 按评价人数降序排列，取前 N 部
- 未登录用户的默认策略
- 推荐理由：`"大家都在看"`

**策略二：内容冷启动 `get_preference_based_recommend()`**

- 读取用户在冷启动问卷中选择的类型偏好
- 70% 来自偏好类型 + 30% 来自其他类型
- 开启探索模式后比例变为 50/50
- 结果随机打散后返回
- 推荐理由：`"因为你的喜好类型：科幻、动作"`

**策略三：NCF 协同过滤 `get_ncf_recommendations()`**

- 加载预训练的 NCF 模型（GMF + MLP 双分支架构）
- 从 Redis 读取用户最近 100 条行为记录
- 构建加权用户向量：点击=0.3 / 收藏=0.7 / 评分=1.0
- 时间衰减：`exp(-0.01 × 距今秒数)`
- 计算用户向量与所有电影 item embedding 的余弦相似度
- 返回相似度最高的 top-K 部电影
- 推荐理由：`"因为你喜欢《星际穿越》"`（取用户历史中权重最高的电影）

## 新增功能（毕业设计加强）

### 1. 推荐结果解释

每部推荐电影附带一句解释文案，提升用户体验：

| 推荐策略 | 解释文案 |
|---------|---------|
| 热门推荐 | "大家都在看" |
| 偏好推荐 | "因为你的喜好类型：科幻、动作" |
| NCF 推荐 | "因为你喜欢《星际穿越》" |

- 实现：后端推荐函数生成 `reason` 字段，前端以灰色斜体小字显示在电影卡片上
- 涉及：首页实时推荐、排行榜 NCF 推荐榜、猜你喜欢

### 2. 电影相似影片推荐

在电影详情页底部展示与当前电影最相似的 5 部影片：

- 基于 NCF 模型的 item embedding 计算余弦相似度
- 实时计算，无需离线预处理
- 标题显示 `"和《XXX》相似的影片"`
- 降级方案：若电影不在模型范围内，回退到热门推荐
- 额外提供管理命令 `python manage.py compute_item_sim` 可离线预计算相似度矩阵并存入数据库

### 3. 用户画像标签加强

个人中心展示**真实计算的**个性化画像，不再使用随机标签：

- **偏好类型 Top3**：基于点击/收藏/搜索过的电影类型统计频次
- **平均评分**：从 MovieRating 表真实聚合
- **活跃等级**：轻度影迷 (< 30) / 中度影迷 (30~100) / 重度影迷 (> 100)
- **近期最爱影片**：Redis 行为记录中权重最高的电影
- **趣味称号**：如 "宇宙探索者"、"悬疑探长"、"动作达人" 等

### 4. 用户反馈循环（实时向量更新）

行为发生后立即更新 Redis 中的用户向量快照，实现近实时兴趣跟踪：

- 向量更新公式：`V_new = V_old × exp(-0.01 × Δt) + w × E_m`
- 权重：点击=0.3 / 收藏=0.7 / 评分=1.0
- 推荐时优先使用缓存向量，避免每次遍历 100 条历史
- 版本号机制：模型重训后自动清空旧向量

### 5. A/B 测试框架

完整的推荐策略对比实验系统：

- **用户分组**：注册/首次登录时随机分配 A 组（对照）或 B 组（实验）
- **策略差异**：B 组在 NCF 推荐中注入 30% 随机多样性电影
- **行为追踪**：RecEvent 表记录每次点击/收藏/评分/展示，附带分组信息
- **个人中心展示**：可视化柱状图对比 A/B 组人均收藏数（Chart.js）
- **数据 API**：`/expstats/` 返回 JSON 格式的实验统计数据

## 页面功能

### 首页 `/` `/index/`

- 3 张轮播图（硬编码）
- 实时推荐 10 部电影（已登录显示个性化推荐 + 推荐理由，未登录显示热门）
- 猜你喜欢 5 部（热门电影，附推荐理由）
- 去重 + 数量补齐机制（确保始终显示 10 部不重复的电影）
- 顶部导航栏：首页 / 分类 / 排行榜 / 我的 / 搜索框
- 无需登录可访问（内容不同）

### 分类页 `/category/`

- 左侧题材列表（19 种，英文→中文翻译）
- 右侧电影网格，每页 15 部
- 排序切换：按年份 / 按评分
- 分页：上一页 / 下一页 / 指定页码跳转
- Ajax 无刷新加载
- 需登录

### 排行榜 `/rank/`

- **评分榜**：平均评分最高的 50 部
- **热度榜**：统计全站用户的点击 + 收藏次数，Top 50
- **NCF 推荐榜**：基于用户行为的个性化推荐 Top 50，显示推荐度和推荐理由
- **新片榜**：占位页面（暂无数据）
- 每部电影显示：排名、海报、标题、年份、评分、星星、简介、操作按钮
- 整页 Redis 缓存 5 分钟
- 需登录

### 详情页 `/detail/<movie_id>/`

- 电影海报、标题、年份、类型、评分
- 剧情简介（硬编码）
- 收藏按钮（Ajax 提交）
- 评分五星弹窗（Ajax 提交，实时更新均分）
- **底部相似影片推荐**（5 部，基于 NCF item embedding 余弦相似度）
- 每次访问自动记录点击行为：计分 +1、点击历史、浏览记录、用户向量增量更新
- 需登录

### 搜索页 `/search/<关键词>/`

- 按标题模糊搜索（LIKE %关键词%）
- 搜索结果全部自动加入收藏队列（与手动收藏同权重）
- 搜索行为纳入 A/B 实验追踪
- 前端支持按年份/评分排序
- 需登录

### 个人中心 `/profile/`

- **账户概览**：
  - 收藏数 / 看过数 / 评价数
  - 真实个性化画像标签（偏好类型 Top3、平均评分、活跃等级、近期最爱、趣味称号）
- **我的收藏**：收藏电影列表
- **浏览记录**：最近 50 条点击记录（保留重复）
- **安全设置**：修改昵称/密码（需验证旧密码）
- **推荐引擎配置**：
  - 探索模式开关（增加推荐多样性）
  - 周一邮件推送开关
- **A/B 实验数据卡片**（新增）：
  - 显示用户所属实验组
  - 个人点击/收藏统计
  - Chart.js 柱状图 → A 组 vs B 组人均收藏数对比
- 需登录

### 登录 `/login/`

- 用户名 + 密码登录
- 记住我（14 天保持登录）
- 首次登录自动分配 A/B 实验分组
- Vue.js 实现表单交互和密码显隐切换

### 注册 `/register/`

- 用户名 + 邮箱 + 密码 + 确认密码
- 邮箱验证码（163 SMTP 发送，5 分钟有效）
- 60 秒发送倒计时
- Vue.js + Axios 实现

### 冷启动问卷（注册后首次登录）

- 选择喜欢的电影类型（多选）
- 设置最低可接受评分（滑块）
- 选择用户活跃等级（下拉）
- 提交后跳转首页，后续推荐基于此偏好

## 用户行为追踪

| 行为 | 触发 | Redis | 数据库 | 向量更新 | A/B 记录 |
|------|------|-------|--------|---------|---------|
| 点击电影详情 | 访问详情页 | `click` | point_count+1 / clicked_movie_ids / MovieWatchRecord | weight=0.3 | RecEvent |
| 收藏电影 | 收藏按钮 | `collect` | collected_movie_ids | weight=0.7 | RecEvent |
| 搜索电影 | 搜索页 | `collect`（每个结果） | collected_movie_ids | - | RecEvent |
| 评分 (1-5星) | 详情页评分弹窗 | `rating` | MovieRating | weight=1.0 | RecEvent |

## 自动化功能

| 功能 | 触发 | 说明 |
|------|------|------|
| 邮箱验证码 | 注册页请求 | 6 位随机数字，Redis 存储 5 分钟有效 |
| 手动邮件推荐 | 个人中心勾选推送 | 提取用户浏览+收藏的 10 部电影，发送 HTML 邮件 |
| 周一自动推送 | APScheduler 定时 | 每周一 8:00，向开启推送的用户发送 NCF 个性化推荐 |
| 排行榜页面缓存 | 每次访问 | Redis 缓存整页数据 5 分钟 |
| 用户向量增量更新 | 每次行为 | 实时维护 Redis `user_vec:{uid}` 快照 |

## 数据存储

| 存储 | 数据库 | 用途 |
|------|--------|------|
| MySQL | `movies_rec` | 电影信息、用户信息、评分、浏览记录、A/B 实验事件 |
| Redis | db0 | 默认缓存 |
| Redis | db1 | Session 会话 |
| Redis | db2 | 邮箱验证码 |
| Redis | db3 | 用户行为历史（最近 100 条）+ 用户向量快照 |

### 核心数据表

| 表 | 主要字段 |
|------|------|
| `movies` | movie_id, title, year, genres, avg_rating |
| `user` | 继承 Django AbstractUser，扩展 email |
| `userprofile` | 偏好设置、点击/收藏历史（JSON 数组）、计分、用户向量缓存、rec_group（A/B 分组） |
| `moviewatchrecord` | user, movie_id, watch_time |
| `movierating` | user, movie_id, rating (1-5) |
| `ratings` | MovieLens 原始评分数据（训练用） |
| `rec_event` | user, movie, event_type, rec_group, source, created_at（A/B 实验记录） |
| `movies_itemsim` | movie_id, sim_movie_id, similarity（可选，离线预计算相似度） |
